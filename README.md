# Azure Cloud Security Implementation

## Project Overview

This project demonstrates the implementation of security controls and governance practices within a Microsoft Azure cloud environment.

The objective was to strengthen cloud security posture through identity management, access control, encryption, monitoring, and compliance-focused governance.

---

## Objectives

- Secure Azure resources using RBAC
- Implement Multi-Factor Authentication (MFA)
- Protect sensitive data using Azure Key Vault
- Improve cloud governance and compliance
- Reduce risk through security monitoring

---

## Scope

The implementation focused on:

- Azure Active Directory (Entra ID)
- Role-Based Access Control (RBAC)
- Multi-Factor Authentication (MFA)
- Azure Key Vault
- Azure Security Center
- Azure Monitor

---

## Security Controls Implemented

### Identity & Access Management

- Role-Based Access Control (RBAC)
- Least Privilege Access
- Conditional Access Policies
- Multi-Factor Authentication

### Data Protection

- Encryption at Rest
- Encryption in Transit
- Azure Key Vault Secret Management

### Monitoring & Logging

- Azure Monitor
- Activity Log Monitoring
- Security Alerts

---

## Risk Reduction Outcomes

| Risk | Mitigation |
|--------|--------|
| Unauthorized Access | MFA + RBAC |
| Credential Theft | Conditional Access |
| Data Exposure | Encryption + Key Vault |
| Lack of Visibility | Azure Monitor |
